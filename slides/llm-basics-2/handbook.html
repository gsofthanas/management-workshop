AIを使う際の基本

この5原則を使い倒します。コードはAIに書かせます。あなたは「何を作るか」と「困ったときの質問」だけ。

  1. 整理する 何が欲しいかを自分の中で明確にしてから依頼する。
  2. 具体的に伝える 曖昧を捨て、条件・制約・形式を明示する。
  3. 構造化する ### 見出し でブロックを分けるとAIが理解しやすい。
  4. AIに先に質問させる プロンプトに自信がないなら「先に5つ質問して」と頼む。AIが要件を引き出してくれる。
  5. わからないことはAIに聞く 用語・概念・手順・エラー — 何でも遠慮なく。

毎ステップ共通

動いたら、AIに「このコードの設計書をMarkdownで書いて」と頼んで保存。次のStepでAIに渡す土台になる。

Step 1: HTMLを作ってみよう

テキストを貼り付けると文字数・読了時間・段落数・キーワードを表示するHTMLツールをAIに作らせます。

AIへの最初の依頼

そのままコピペでOK 「ササッと読む」というブラウザで動く文章解析ツールを作りたい。 要件を整理するために、まず私に5つだけ質問してください。 答えてからコードを書き始めてください。

AIから返ってくる質問の例 と お手本回答

こんな感じで対話が進みます。自分の好きな答えに変えてもOK。

テキストはどう入力しますか？
テキストエリアに貼り付ける形式
どんな解析項目が必要ですか？
文字数、読了時間、段落数、英単語タグ、カタカナ語タグ
結果の表示方法は？
解析ボタンを押した後にカード形式で表示
デザインの雰囲気は？
白基調でモダン、角丸
外部ライブラリは？
使わない。HTML/CSS/JSの1ファイル完結

完成チェック

  • HTMLファイルをダブルクリックでブラウザに開ける
  • テキストを貼り付けて「解析する」ボタンで結果が出る
  • 英単語・カタカナ語のタグが頻度順に並んでいる

完了したら

AIに 「このHTMLの設計書をMarkdownで書いて」 と依頼。Step 2 でAIに渡す土台になる。

Step 2: 拡張機能を作ってみよう

Step 1 のHTMLを Chrome 拡張機能にして、任意のWebページから本文を取得できるようにします。

AIへの依頼

Step 1 の設計書と一緒に渡す このHTMLツールを Chrome 拡張機能（Manifest V3）にしたい。 - 「📥 このページから取得」ボタンを popup に追加 - ボタンを押すと現在のタブの本文を抽出してテキストエリアに入れる - Wikipedia / ニュース / 一般記事 で動くようにする manifest.json / popup.html / popup.js / content.js を作ってください。 迷う点があれば先に質問してください。

拡張機能を Chrome に読み込む手順

これだけは AI ではなく、自分の手で操作します。

  1. chrome://extensions をアドレスバーに入力
  2. 右上の「デベロッパーモード」をオン
  3. 「パッケージ化されていない拡張機能を読み込む」をクリック
  4. AIが作ったファイルを置いたフォルダを選択
  5. ツールバーの拡張アイコンから起動

完成チェック

Wikipedia や ニュース記事を開いて、拡張アイコン → 「ページから取得」 で本文がテキストエリアに入れば成功。

完了したら

AIに 「この拡張機能の設計書をMarkdownで書いて」 と依頼。Step 3 でAIに渡す土台になる。

Step 3: AI機能を追加してみよう

拡張機能から Anthropic Claude API を呼び出して、本文を3行に要約します。

⚠️ APIキーの大原則

APIキーは chrome.storage.local にだけ保存。ソースコードに絶対書かない。Gitに上げない。Slack・メール・画面共有も全部NG。流出すると一晩で数百万円の被害になる事例あり。

AIへの依頼

Step 2 の設計書と一緒に渡す この拡張機能に「✨ 3行で要約」ボタンを追加したい。 - ボタンを押すとテキストエリアの内容を Anthropic Claude API に送る - 結果を3行（結論 → 理由 → 使い所）で表示する - APIキーは設定UIから入力させて chrome.storage.local に保存 - ソースコードに絶対書かない - manifest.json に必要な権限を追加 迷う点があれば先に質問してください。

システムプロンプトのヒント

「3行要約のシステムプロンプトを作って」と AI に頼むときは、こんな要素を含めるよう指定すると精度が上がります。

  • 役割 — 「あなたは記事を3行で要約するエキスパートです」
  • 出力の型 — 結論 / 理由 / 使い所 の順
  • 文字数制限 — 各行40文字以内
  • 出力フォーマット — 番号や記号なし、改行のみ

完成チェック

ニュース記事を開いて「ページから取得」→「3行で要約」で、3行の要約が表示されたら成功。

完了したら

AIに 「完成版ツールの設計書をMarkdownで書いて」 と依頼。明日からの自分のリファレンスになる。

ボーナス課題

最低限が動いたらこれらを試そう。全部 AI に頼めば実装できる。

モードを増やす

システムプロンプトを変えるだけで別のモードが作れる。AIに「○○モードのシステムプロンプトを書いて」と頼む。

  • やさしい日本語モード — 専門用語を中学生にもわかる言葉に
  • 翻訳モード — 英語ページを自然な日本語に
  • TODO抽出モード — 議事録からアクションアイテムだけ抽出

機能を追加する

AIに「○○機能を追加して」と個別に頼めば、既存コードに足してくれる。

  • 履歴機能 — 過去の要約を chrome.storage.local に保存
  • コピーボタン — 要約結果をクリップボードにコピー
  • 読み上げ機能 — 音声で読み上げ（Web Speech API）
  • ダークモード — 目に優しい配色

トラブルシューティング

用語や概念がわからない

その場でAIに聞いてください。これがこの研修の重要な学びです。

質問例 manifest.json って何？初心者にもわかるように 5 行で説明して。 content script と popup script の違いを、子供に説明するように教えて。 CORS って何？なぜ起きるの？

AIが出したコードがエラーになる

  • エラーメッセージをそのままコピーして AI に貼る
  • 「このエラーが出ました。原因と修正方法を教えて」と聞く
  • 何度かやり取りすればだいたい直る

拡張機能が読み込めない

  • chrome://extensions でエラーメッセージを確認
  • そのエラーをAIに貼って「直して」と頼む
  • ファイル名が manifest.json になっているか（.txt が付いていないか）確認

「ページから取得」が動かない

  • そのサイトのURLを AI に教えて「このサイトから本文を取れるようにして」と頼む
  • セレクタ調整は AI の得意分野

APIキーエラー

  • キーが正しく保存されているか設定UIで確認
  • キーの先頭が sk-ant-api03- で始まっているか
  • 余分なスペースや改行が混じっていないか

要約結果が空っぽ

  • レスポンスを console.log で表示してAIに貼る
  • 「このレスポンスから要約テキストを取り出すには？」と聞く
